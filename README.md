📄 README: Подписатор XML/GGE (Авто-выбор) v4.0
Подписатор 10000 — это графический инструмент на базе PowerShell с интерфейсом Windows Forms, предназначенный для автоматизации массового подписания XML и GGE (Главгосэкспертиза) файлов с использованием утилиты КриптоПро CSP (cryptcp.exe). 
Скрипт автоматически извлекает ФИО подписантов из структуры XML-файлов, сопоставляет их с сертификатами, установленными в системе, и генерирует файлы подписей.
🚀 Основные возможности

    Drag & Drop: Удобное перетаскивание файлов или папок прямо в окно программы.
    Универсальный парсер XML: Автоматически находит узел <Signatures> и извлекает ФИО из ролей (Composer, Verifier, ComposeFIO, VerifyFIO и др.).
    Авто-выбор сертификатов: Сопоставляет фамилии из XML с именами владельцев сертификатов в хранилище Windows (Личное / My).
    Гибкая привязка: Каждый файл подписывается только теми сертификатами, которые указаны в его структуре (поддерживается многократное подписание).
    Интерактивный UI: Таблица с результатами парсинга, поиск по сертификатам, прогресс-бар и логирование.
    Автоматическое именование: Подписи сохраняются в ту же папку с понятным именем, содержащим ФИО подписанта.

⚙️ Системные требования

    ОС: Windows 10 / 11 / Server.
    PowerShell: версии 5.1 или выше (встроен в Windows).
    КриптоПро CSP: Установленная лицензия и утилита командной строки cryptcp.exe.
        Скрипт автоматически ищет её по путям:
            C:\Program Files\Crypto Pro\CSP\cryptcp.exe
            C:\Program Files (x86)\Crypto Pro\CSP\cryptcp.exe
            взять можно здесь https://cryptopro.ru/products/csp/downloads
    Сертификаты: Необходимые сертификаты с закрытым ключом должны быть установлены в личное хранилище текущего пользователя (CurrentUser\My).

📖 Инструкция по использованию
Шаг 1. Запуск
Запустите скрипт в PowerShell. Откроется графическое окно программы.
Шаг 2. Загрузка файлов
Перетащите XML/GGE файлы в синюю зону вверху окна или кликните по ней для выбора через диалоговое окно.

    В таблице отобразятся файлы, роли (Составитель/Проверяющий) и распознанные ФИО.

Шаг 3. Авто-выбор сертификатов
Нажмите кнопку «АВТО-ВЫБОР ПО ФАЙЛАМ».

    Скрипт просканирует личное хранилище сертификатов.
    По фамилии из XML он найдет соответствующий сертификат.
    В таблице в колонке «Сертификат» появится имя владельца (зеленым цветом) или «Не найден» (красным).
    В списке сертификатов внизу автоматически проставятся галочки.

Шаг 4. Ручная корректировка (опционально)
Если авто-выбор сработал неверно (например, в системе два сертификата с одной фамилией), вы можете:

    Использовать Поиск для фильтрации списка сертификатов.
    Вручную снять/поставить галочки в списке CheckedListBox.
    (Примечание: ручная привязка конкретного сертификата к конкретной строке таблицы в текущей версии не предусмотрена, авто-выбор работает по глобальному списку выбранных галочками сертификатов).

Шаг 5. Подписание
Нажмите кнопку «ПОДПИСАТЬ ВЫБРАННЫЕ».

    Подтвердите операцию во всплывающем окне.
    Дождитесь завершения процесса (отображается в прогресс-баре).
    По окончании появится отчет об успешно созданных файлах и ошибках.

🧠 Технические детали и логика работы
1. Логика парсинга ФИО из XML
Скрипт ищет блок <Signatures>. Внутри него перебираются дочерние узлы (роли).

    Если внутри роли есть тег <Name>, берется его содержимое.
    Если тега нет, берется весь текст внутри узла роли.
    Алгоритм выделения Фамилии: Строка разбивается по пробелам. Фамилией считается самое длинное слово, не содержащее точек (чтобы отсечь инициалы И.И.). Если все слова с точками, берется просто самое длинное.

2. Логика именования выходных файлов
Файлы подписей создаются в той же директории, что и исходные XML/GGE.

    Формат имени: ИмяБезРасширения_ФИО_Подписанта.расширение.sig
    Пример: Договор_Иванов Иван Иванович.xml.xml.sig (длинное ФИО обрезается до 40 символов для избежания ошибок ОС).
    Примечание: cryptcp сначала создает временный файл .sgn, который затем скрипт переименовывает в .sig. Если файл с таким именем уже существует, к нему добавляется суффикс _1, _2 и т.д.

3. Карта подписания (FileCertMap)
Скрипт строит внутреннюю карту: Файл -> [Список отпечатков (Thumbprint) сертификатов]. При нажатии кнопки "Подписать" цикл идет строго по этой карте, что гарантирует, что файл не будет подписан "чужими" сертификатами, которые просто стоят галочками в общем списке.
